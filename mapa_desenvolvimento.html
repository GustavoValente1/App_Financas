Descoberta

Ideação & PRD

Bate-papo socrático com Claude para mapear o problema, definir o produto e tomar todas as decisões antes de escrever qualquer código.

Método do Bate-papo
  • Iniciar descrevendo o problema atual (planilha Excel com dashboard quebrado e sem acesso mobile)
  • Claude faz perguntas uma de cada vez — usuário responde de forma concisa
  • Cada resposta desbloqueia a próxima decisão de produto
  • Mockups visuais gerados e aprovados no chat, sem código
  • Todas as decisões documentadas com o "porquê" explícito
  • Output final: PRD.md + Histórico completo da conversa
Decisões Tomadas
Multi ou single user?
→ Usuário único — sem multiusuário, sem permissões
Onde ficam os dados?
→ Nuvem (Supabase) — sync celular ↔ desktop
Velocidade de lançamento?
→ 2 toques: subcategoria + valor. Resto automático
Como calcular o "quanto gastamos"?
→ Por competência (não caixa)
Gráficos em aba separada?
→ Não — na tela principal, na rolagem
Quais telas?
→ Dashboard, Novo Lançamento, Lançamentos, Cadastros
Outputs
📄 PRD.md 💬 Histórico_Conversa.md ✅ 6 decisões aprovadas 🎨 Mockups aprovados no chat

Implementação

Desenvolvimento Iterativo

Cada feature segue o mesmo ciclo: spec → código → commit → push. 13 commits do zero ao app completo com todas as features.

Ciclo por Feature
  • 1. Spec MD: descrever o comportamento esperado antes de codar
  • 2. Código: Claude Code implementa seguindo a spec
  • 3. Revisão: testar no browser, ajustar se necessário
  • 4. Commit: conventional commit com mensagem descritiva
  • 5. Push: imediato após o commit (regra do CLAUDE.md)
Features do App
  • Dashboard com HeroCard, gráficos e resumo do mês
  • NewTransactionSheet — bottom sheet com lançamento rápido
  • Aba Lançamentos com filtro combinado e edição
  • Aba Cadastros (categorias, fontes, origens de receita)
  • Relatórios: Fluxo de Caixa + Conferência de Extrato
  • Period selector + toggle competência / caixa
  • Clone de transação + Import/Export Excel
  • Sort por data de pagamento e competência

05
Qualidade

Padrões & Boas Práticas

As práticas que tornaram o desenvolvimento fluido e o resultado previsível — aplicáveis a qualquer projeto novo.

Práticas de Desenvolvimento
  • Spec-first: escrever arquivo MD descrevendo a feature antes de codar — nunca ao contrário
  • Conventional commits: feat: fix: docs: refactor:
  • Auto-commit+push: via CLAUDE.md — nenhuma alteração fica sem histórico
  • Screenshots como referência: imagens do produto de referência na raiz do projeto
  • Seed com dados reais: banco pré-populado com dados da planilha original desde o início
Configuração do Claude Code
  • CLAUDE.md no projeto: instruções que o Claude segue em toda sessão, sem precisar repetir
  • Memory files: contexto persistente entre sessões em ~/.claude/projects/
  • Tipos de memória usados: project (contexto do app), feedback (comportamentos aprovados), user (perfil e preferências)
  • PRD como contexto: enviar PRD.md e histórico da conversa ao iniciar o desenvolvimento no Claude Code
  • Docs de specs versionadas: cada spec.md commitada junto com o código que implementa
Resultado
🧠 Claude sempre em contexto 📂 Histórico limpo e legível 🔁 Processo replicável 🚫 Zero retrabalho por falta de spec
